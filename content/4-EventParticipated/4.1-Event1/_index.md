# Reflection on “FCAJ Community Day”

### Event purpose

This event was designed not only to introduce new technology topics, but also to help students and freshers rethink how they learn, work, and use AI effectively in software development. The main themes included:

- Sustaining long-term learning motivation
- Prompt Engineering for better AI output
- Professional mindset, foundation, and integrity
- The BMX Method for AI-assisted software development

### Speakers

- **Mr. Khang**: Shared insights on learning motivation and career mindset
- **Unidentified speaker**: Presented Prompt Engineering and demonstrated an AI serverless project on AWS
- **Mr. Thao**: Software Developer at VIB, introduced the BMX Method

### Key highlights

#### 1. Learning motivation and overcoming procrastination

One of the most practical points in the event was the explanation of why people often lose focus while studying. Social media and online entertainment tend to provide quick dopamine rewards, while learning requires patience before results become visible. Because of that, maintaining a study routine needs intentional structure rather than motivation alone.

Suggested approaches included:

- Building a daily learning streak
- Breaking large goals into smaller tasks
- Creating small rewards after each study session

#### 2. Prompt Engineering and working with AI

The Prompt Engineering section helped me understand that AI output quality depends heavily on how the request is framed. A strong prompt should clearly define role, task, context, input, example, output format, and constraints.

The speaker also emphasized several useful practices:

- Provide enough context instead of asking vague questions
- Separate ideas into clear sections
- Avoid mixing too many unrelated topics in one chat flow
- Always review the result to reduce hallucination

The AWS serverless demo gave me a more concrete view of how CloudFront, S3, API Gateway, Lambda, Amazon Bedrock, DynamoDB, and Cognito can work together in a complete AI application.

#### 3. Professional mindset in the AI era

A takeaway that stayed with me was the idea that AI does not simply replace developers. Instead, it amplifies whatever foundation a person already has. Strong fundamentals lead to better outcomes with AI, while weak fundamentals create bigger mistakes faster.

From that perspective, three points stood out:

- Always ask “Why?” instead of focusing only on “What?”
- Maintain integrity, especially when handling edge cases
- Evaluate work opportunities by experience, network, and long-term growth, not salary alone

#### 4. BMX Method and AI in the SDLC

The BMX Method changed the way I think about using AI for coding. Instead of prompting randomly and accepting raw output, this method starts with clear documentation, divides work into Epic, Story, and Sub-task, and assigns specific responsibilities to AI roles such as PM, Architect, Scrum Master, Developer, and Reviewer.

This approach helps:

- Reduce disconnected or low-context code generation
- Keep quality under control as the system grows
- Prevent AI roles from overlapping too much

### What I learned

#### Design and programming mindset

- A strong foundation matters more than chasing many popular frameworks
- AI should be used within a structured design process, not as a replacement for engineering judgment
- Prompting is also a kind of architecture for communication

#### Technical perspective

- I gained a clearer picture of how a serverless GenAI system can be built on AWS
- I learned to structure prompts more carefully for daily technical work
- I saw the importance of documentation when collaborating with AI tools

#### Career strategy

- At the fresher stage, a learning-rich environment is extremely valuable
- Mistakes are acceptable if I can trace the root cause and fix them responsibly
- Long-term growth is often more meaningful than short-term benefits

### Applying this to my work

After the event, I planned to apply several ideas immediately:

- Write more structured prompts when using AI for coding tasks
- Draft documents and architecture before asking AI to generate code
- Maintain a daily learning or commit streak
- Practice asking “why” behind each technical pattern or decision

### Event experience

For me, “FCAJ Community Day” was more than a technical sharing session. It gave me both practical tools and a clearer learning mindset for working in the AI era. The biggest value came from seeing how technical skill, discipline, and long-term thinking can reinforce each other in a real career path.
